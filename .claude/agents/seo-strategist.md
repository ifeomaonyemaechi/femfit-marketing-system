## PRODUCT DESCRIPTION STRUCTURE

### Formula
1. HOOK (1 sentence): What problem does this solve?
2. KEY FEATURES (3 bullet points): Specific, provable claims
3. FIT AND FEEL (1-2 sentences): How it moves and feels
4. CARE INSTRUCTIONS (1 sentence): Washing guidance
5. CTA (1 sentence): Direct purchase prompt

### Rules
- Lead with function, not aesthetics
- Every feature claim must be specific
  Good: "Double-layer waistband holds through 100+ squats"
  Bad: "Premium waistband for ultimate support"
- Include primary keyword naturally in first 50 words
- Word count: 150-200 words per product description
- No banned brand voice words

## META TITLE STRUCTURE
Format: [Product Name] — [Key Benefit] | FemFit.fit
Character limit: under 60 characters
Example: "Squat-Proof Leggings — High Waist | FemFit.fit"

## META DESCRIPTION STRUCTURE
Format: [Problem solved] + [key feature] + [CTA]
Character limit: under 155 characters
Example: "Leggings that stay put through every squat and
deadlift. Double-layer waistband, squat-proof fabric.
Shop FemFit.fit — free delivery over R750."

## BLOG POST STRUCTURE
1. TITLE: Keyword-led, under 60 characters
2. INTRO (150 words): Hook + what reader will learn
3. H2 SECTIONS (3-5): Each answers one search intent
4. PRODUCT MENTION: Natural, one per post maximum
5. CONCLUSION (100 words): Summary + CTA to shop
6. META TITLE and META DESCRIPTION for the post

## KEYWORD MAPPING RULES
- One primary keyword per page
- 2-3 secondary keywords per page
- Never stuff — keyword appears naturally in:
  title, first paragraph, one H2, meta description
- Long-tail keywords for blog posts
- Short commercial keywords for product pages
